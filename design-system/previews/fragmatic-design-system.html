04 — Foundations

Borders.

Border roles are semantic foundation tokens. Use these names for passive edges, hoverable controls, focus affordances, selected states, and danger states instead of local color picks.

Subtle edge

Default page, card, divider, and shell borders.

--border-subtle · --border-width-default
Strong edge

Hoverable controls and surfaces that need a clearer edge.

--border-strong
Interactive edge

Reusable hover border for buttons, field chips, dropdowns, and cards.

--border-interactive · --frgm-effect-hover-border
Focus edge

Keyboard-visible and input focus treatments.

--border-focus · --frgm-control-focus-ring
Selected edge

Active selection, chosen items, and context navigation state.

--border-selected · --frgm-state-selected-border
Danger edge

Validation, destructive choices, and error-adjacent controls.

--border-danger · --frgm-state-danger-fg
05 — Foundations

Effects, motion.

Effects and motion are foundation recipes. Components should consume transition, hover, focus, elevation, and loading tokens rather than inventing one-off animation timings or shadows.

Transition recipe

Shared property set for background, border, text, and shadow changes.

Hover to inspect transition
--frgm-effect-transition
Hover effect

Reusable hover background, border, shadow, and one-pixel lift.

Hover surface
--frgm-effect-hover-* · --frgm-motion-hover-y
Elevation

Flat, raised, and popover surfaces use named shadow tokens.

Popover shadow
--shadow-soft · --shadow-elev · --shadow-pop
Timing

Fast, medium, slow, enter, exit, spin, and typing recipes.

0.12s / 0.15s / 0.24s
--motion-fast · --motion-med · --frgm-motion-*
